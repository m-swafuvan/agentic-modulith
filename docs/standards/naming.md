# Naming Standards

# Philosophy

Naming should optimize for clarity, maintainability, and architectural consistency.

---

# Rules

Prefer:

- explicit names
- domain terminology
- consistent vocabulary
- descriptive intent

Methods SHOULD use verbs.

Entities SHOULD use nouns.

---

# Avoid

Avoid vague naming such as:

- Helper
- Util
- Manager
- Common
- Processor

Names should communicate responsibility clearly.

---

# Consistency Rules

Naming SHOULD remain:

- predictable
- domain-oriented
- readable
- implementation-aware

Consistency is more important than creativity.
